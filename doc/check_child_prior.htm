Check Priors in Child Smoothing

syntax
check_child_prior(
     db, rate_table, smooth_grid, nslist_pair, prior_table
)

Purpose
Checks the child smoothing assumptions .
  1. The density_id cannot correspond to a laplace or log_laplace density.
  2. The lower limit must be minus infinity or equal to the upper limit and finite.
  3. The upper limit must be plus infinity or equal to the lower limit and finite.


db
This argument has prototype
     sqlite3* db
 and is the database connection for logging errors.

rate_table
This argument has prototype
     const CppAD::vector<rate_struct>& rate_table
 and it is the rate_table . For this table, only the child_smooth_id field is used.

smooth_grid
This argument has prototype
     const CppAD::vector<smooth_grid_struct>& smooth_grid
 and it is the smooth_grid . For this table, only the value_prior_id, dage_prior_id, and dtime_prior_id fields are used.

nslist_pair
This argument has prototype
     const CppAD::vector<nslist_pair_struct>& nslist_pair
 and it is the nslist_pair .

prior_table
This argument has prototype
     const CppAD::vector<prior_struct>& prior_table
 and it is the prior_table . For this table, only the value_prior_id, dage_prior_id, and dtime_prior_id, field are used.
