Put a Message in the Log Table

Syntax
db
os
message_type
message
table_name
row_id
unix_time
Example

Syntax
unix_time = log_message(db, os, message_type, message)
 unix_time = log_message(
     db, os, message_type, message, table_name, row_id
)

db
This argument has prototype
     sqlite3* db
 and is the database connection.

os
This argument has prototype % std::ostream* %os% % If it is not null and message_type is error or warning, the message is also written to the output stream with a new line added at the end (as well as being logged in the database).

message_type
This argument has prototype
     const std::string& message_type
 and must be one of the following: command, error, warning, value. This value gets written in the message_type column of the log table.

message
This argument has prototype
     const std::string& message
 This value gets written in the message column of the log table.

table_name
This argument has prototype
     const std::string& table_name
 This value gets written in the table_name column of the log table. If table_name it is not present or empty, null is used.

row_id
This argument has prototype
     const size_t& row_id
 This value gets written in the row_id column of the log table. Note that the value DISMOD_AT_NULL_SIZE_T gets converted to a null. If row_id is not present, null is used. If row_id is present and not DISMOD_AT_NULL_SIZE_T, table_name must not be empty.

unix_time
The return value has prototype
     std::time_t unix_time
 It is the value written in the log table for unix_time .

Example
Check the log table in the database after any command .
