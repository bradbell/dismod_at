Data Likelihood and Weighted Residuals

Data Table Notation

i
We use @(@ i @)@ to denote the data_id for a row in the data table.

y_i
We use @(@ y_i @)@ to denote the corresponding meas_value value.

a_i
We use @(@ a_i @)@ to denote the corresponding age_lower value.

b_i
We use @(@ b_i @)@ to denote the corresponding age_upper value.

s_i
We use @(@ s_i @)@ to denote the corresponding time_lower value.

t_i
We use @(@ t_i @)@ to denote the corresponding time_upper value.

d_i
We use @(@ d_i @)@ to denote the density_id and eta value corresponding to @(@ i @)@.

I_i
We use @(@ I_i @)@ to denote the corresponding integrand_id value.

x_ij
We use @(@ x_{i,j} @)@ to denote the corresponding covariate values where @(@ j @)@ denotes the covariate_id . Note that the covariate reference has already been subtracted from these values.

w_i
We use @(@ w_i (a, t) @)@ for the weighting as a function of age and time that corresponds to the weight_id for this data_id .

Delta_i
Let minimum_meas_cv be the integrand table minimum_meas_cv corresponding to this integrand @(@ I_i @)@. Let meas_std and meas_value correspond to this data table data_id . We use the notation @(@ \Delta_i @)@ for the modified standard deviation used for this data
     Delta_i = max(meas_std, minimum_meas_cv * |meas_value| )
Measurement Standard Deviation Covariates

K_i
There is a set of rows in the mulcov table such that integrand_id is equal to @(@ I_i @)@. and mulcov_type is equal to meas_std. We use @(@ K_i @)@ to denote the corresponding set of covariate_id values for which this is the case.

gamma_j
For each covariate index @(@ j \in K_i @)@ , we use @(@ \gamma_j (a, t) @)@ to denote the piecewise linear function corresponding to the covariate multiplier variables for the corresponding mulcov_id . Note that these are only meas_std covariate multipliers.

Effects, u, theta
We use @(@ u @)@ and @(@ \theta @)@ to denote the vector of random effects and fixed effects respectively. There is a different random effects vector @(@ u @)@ for each child. The node_id for each data point determines the random effects for child data . Note that there are no random effects for parent data . This corresponds to @(@ u = 0 @)@.

Average Integrand
Let @(@ A_i ( u , \theta ) @)@ denote the average integrand for this data_id .

Adjusted Standard Deviation, delta_i
We define the standard deviation effect by @[@ E_i ( \theta ) = \frac{1}{\bar{w}_i} \int_{a(i)}^{b(i)} \int_{s(i)}^{t(i)} \left( \sum_{j \in K_i} x_{i,j} \gamma_j (a, t) \right) w_i (a,t) \; \B{d} t \; \B{d} a @]@ For the log-transformed densities, the adjusted standard deviation is @[@ \delta_i ( \theta ) = \Delta_i [ 1 + E_i (\theta) ] @]@ For the other densities, the adjusted standard deviation is @[@ \delta_i ( \theta ) = \Delta_i + E_i (\theta) @]@ Note that for @(@ j \in K_i @)@ it should hold that @(@ 0 \leq x_{i,j} @)@, and @(@ 0 \leq \gamma_j (a, t) @)@ so that @(@ \Delta_i \leq \delta_i ( \theta ) @)@.

Weighted Residual and Likelihood
Given the random effects @(@ u @)@ and the fixed effects @(@ \theta @)@, the weighted residual is @[@ - R [ A_i ( u, \theta ) \W{,} y_i \W{,} \delta_i ( \theta ) \W{,} d_i ] @]@ see the weight residual function R . Note that @(@ y_i @)@ (@(@ \mu @)@) here is plays the role of @(@ \mu @)@ (@(@ y @)@) in the function @(@ R(y, \mu , \delta , d) @)@. This is so that sigma does not depend on the model_variables . Also note the negative sign in front of the weighted residual is so that it has the same sign as @(@ y_i - A_i ( u, \theta ) @)@ (data minus model). The likelihood is @[@ \B{p}( y_i | u , \theta ) = \exp \left( D [ A_i ( u, \theta ) \W{,} y_i \W{,} \delta_i ( \theta ) \W{,} d_i ] \right) @]@ see the log-density function D .
