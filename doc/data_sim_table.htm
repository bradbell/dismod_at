Example
The data_sim table is created during a simulate_command . It contains number_simulate sets of measurements where each set has one value for each entry in the data_subset_table .

data_sim_id
This column has type integer and is the primary key for this table. Its initial value is zero, and it increments by one for each row. Given the model_variables as specified by truth_var_table , the measurement uncertainty is independent for each row and has standard deviation meas_std .

simulate_index
The column has type integer. It specifies the index for this simulated measurement set. This index starts at zero, repeats as the same for the entire subset of data_id values, and then increments by one between measurement sets. The final (maximum) value for simulate_index is number_simulate minus one.

data_subset_id
This column has type integer and is the primary key for the data_subset_table . This identifies which data_id each row of the data_sim table corresponds to. If n_subset is the number of rows in the data_subset table,
     data_sim_id = simulate_index * n_subset + data_subset_id
 for simulate_index equal zero to number_simulate-1 and data_subset_id equal zero to n_subset-1 .

data_sim_value
This column has type real and is the simulated measurement value that for the specified row of the data table; i.e., it can act as a replacement for the value meas_value in the data table. If the density for this data_id is censored (not censored) data_sim_value has value max(z, 0) , ( z ); see z below.

data_sim_delta
The column has type real and is the adjusted standard deviation corresponding to this simulated measurement; see delta_hat below.

data_sim_stdcv
The column has type real and is the minimum cv standard deviation Delta , for use when fitting this simulated data; see Delta_hat below.

Method

data_id
We use data_id to denote the data_id corresponding to the data_subset_id corresponding to this data_sim_id .

y
We use @(@ y @)@ to denote the data table meas_value corresponding to this data_id .

Delta
We use @(@ \Delta @)@ to denote the minimum cv standard deviation corresponding to the data table and this data_id .

d
We use @(@ d @)@ to denote the density_id corresponding to the data table and this data_id .

eta
We use @(@ \eta @)@ to denote the eta corresponding to the data table and this data_id .

A
We use @(@ A @)@ denote the average integrand corresponding to the truth_var table value for the model variables, the values in the data table, and this data_id .

delta
We use @(@ \delta @)@ to denote the adjusted standard deviation corresponding to the truth_var table value for the model variables, the values in the data table, and this data_id .

sigma
We use @(@ \sigma @)@ to denote the transformed standard deviation corresponding to the data table and this data_id ; i.e., @[@ \sigma = \left\{ \begin{array}{ll} \log( y + \eta + \delta ) - \log(y + \eta) & \R{if \; log \; density} \\ \delta & \R{otherwise} \end{array} \right. @]@ Note that @(@ \sigma @)@ does not depend on simulated noise @(@ e @)@ defined below.

e
We use @(@ e @)@ to denote the noise simulated with mean zero, standard deviation @(@ \sigma @)@, and density corresponding to this data_id without log qualification. For example, if the data density for this data_id is log_gaussian, the @(@ e @)@ is simulate using a Gaussian distribution.

z
We use @(@ z @)@ to denote the simulated data value data_sim_value corresponding to this data_sim_id . It the density is linear , @[@ z = A + e @]@ It the density is log scaled , @[@ \begin{array}{rcl} e & = & \log( z + \eta ) - \log( A + \eta ) \\ \exp (e) & = & ( z + \eta ) / ( A + \eta ) \\ z & = & \exp(e) ( A + \eta ) - \eta \end{array} @]@

delta_hat
Given @(@ z @)@, we can solve the equation @[@ \sigma = \log( z + \eta + \hat{\delta} ) - \log( z + \eta ) @]@ to obtain the value @[@ \hat{\delta} = \left\{ \begin{array}{ll} [ exp(\sigma) - 1 ] ( z + \eta ) & \R{if \; log \; density} \\ \sigma & \R{otherwise} \end{array} \right. @]@ The value @(@ \hat{\delta} @)@ is the adjusted standard deviation that corresponds to measurement error @(@ \sigma @)@ and measurement value @(@ z @)@. Note that if @(@ z @)@ happened to be equal to @(@ y @)@, then @(@ \hat{\delta} @)@ would be equal to @(@ \delta @)@.

Delta_hat
Given @(@ \hat{\delta} @)@ we can compute a minimum cv standard deviation @(@ \hat{\Delta} @)@; see adjusted standard deviations .
  1. Adding standard deviations and no scaling: In this case @[@ \begin{array}{rcl} \hat{\delta} & = & \hat{\Delta} + E_i ( \theta ) \\ \hat{\Delta} & = & \hat{\delta} - E_i ( \theta ) \end{array} @]@
  2. Adding standard deviations with scaling: In this case @[@ \begin{array}{rcl} \hat{\delta} & = & \hat{\Delta} [ 1 + E_i ( \theta ) ] \\ \hat{\Delta} & = & \hat{\delta} / [ 1 + E_i ( \theta ) ] \end{array} @]@
  3. Adding variances and no scaling: In this case @[@ \begin{array}{rcl} \hat{\delta} & = & \sqrt{ \hat{\Delta}^2 + E_i ( \theta ) } \\ \hat{\Delta} & = & \sqrt{ \hat{\delta}^2 - E_i ( \theta ) } \end{array} @]@
  4. Adding variances with scaling: In this case @[@ \begin{array}{rcl} \hat{\delta} & = & \hat{\Delta} \sqrt{ 1 + E_i ( \theta ) } \\ \hat{\Delta} & = & \hat{\delta} / \sqrt{ 1 + E_i ( \theta ) } \end{array} @]@
Note that if @(@ z @)@ happened to be equal to @(@ y @)@, then @(@ \hat{\Delta} @)@ would be equal to @(@ \Delta @)@.
