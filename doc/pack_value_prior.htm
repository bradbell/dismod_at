Prior Ids In Same Order As Packed Variables

Syntax
pack_value_prior(
     value_prior_id_out, const_value_out, pack_object, s_info_vec
)

pack_object
This argument has prototype
     const pack_info& pack_object
 and is the pack_info information corresponding to the model_variables .

s_info_vec
This argument has prototype
     const CppAD::vector<smooth_info>& s_info_vec
 For each smooth_id ,
     s_info_vec[ smooth_id ]
 is the corresponding smooth_info information.

value_prior_id_out
This argument has prototype
     CppAD::vector<size_t>& value_prior_id_out
 and on input it's size is zero. Upon return, its size is pack_object.size() and the element value_prior_id[i] is the prior_id corresponding to the i-th model variable in the order determined by pack_object . If
     value_prior_id_out[i] == DISMOD_AT_NULL_SIZE_T
 the i-th model variable is constrained to have value const_value_out[i] .

const_value_out
This argument has prototype
     CppAD::vector<double>& const_value_out
 and on input it's size is zero. Upon return, its size is pack_object.size() and the element const_value_out[i] is the value that the i-th model variable is constrained to. If
     const_value_out[i] == nan
 value_prior_id[i] is not DISMOD_AT_NULL_SIZE_T and it specifies the prior for the value of i-th model variable.

Example
The file pack_value_prior_xam.cpp contains an example and test that uses this routine.
