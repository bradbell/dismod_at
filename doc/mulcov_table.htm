The mulcov Table: Covariate Multipliers
This table specifies the covariate multipliers and their priors. If this table is empty, there are no covariate multipliers in the model. This table has the following columns

mulcov_id
This column has type integer and is its value is the primary key for this table. Its initial value is zero, and it increments by one for each row.

mulcov_type
This column has type text. The possible values are:

rate_value
This covariate multiplier adjusts a rate in the average integrand calculation; see alpha_jk . A separate solution of the ordinary differential equation is required for each cohort and each measurement, that has values not equal to the reference for the covariate specified by covariate_id . In other words, measurements that have the reference value for all the rate covariates do not add as much to the computational load.

meas_value
This covariate multiplier adjusts the average integrand to match the measurement value ; see beta_j .

meas_std
This covariate multiplier adjusts the measurement standard deviation ; see gamma_j .

rate_id
This column has type integer. If mulcov_type is rate_value, rate_id is the rate_id that determines the rate that this covariate and multiplier effects. If mulcov_type is meas_value or meas_std, this column must be null.

integrand_id
This column has type integer. If mulcov_type is meas_value or meas_std, integrand_id is the integrand_id that determines which measurement integrand this multiplier effects. If mulcov_type is rate_value, this column should be null.

covariate_id
This column has type integer and is a covariate_id in the covariate table. This specifies the covariate column in the data_table that this multiplier acts on.

smooth_id
This column has type integer and its value is a smooth_id in the smooth_grid table. It specifies the smoothing uses as a prior for this covariate multiplier. (Note that all covariate multipliers are fixed effects.)

null
If the smooth_id is null, the corresponding covariate multiplier is always zero and no model_variables are allocated for it.

pini
Consider the case where mulcov_type is rate_value and rate_id corresponds to rate_name pini. In this case, smooth_id should correspond to n_age equal to one because variation w.r.t. age will have no effect (for this covariate multiplier).

Example
The files mulcov_table.py and get_mulcov_table_xam.cpp contain example covariate tables.
