The Sample Command

Syntax
dismod_at database sample method number_sample

database
Is an http://www.sqlite.org/sqlite/ database containing the dismod_at input tables which are not modified.

number_sample
Is the number of samples. Each sample contains a complete set of model variables. See the different methods below for more discussion of number_sample .

method
The sample command argument method must be one of the following:

simulate
If method is simulate, number_sample must be equal to number_simulate . The variable sample corresponding to each sample_index the sample table is the optimal estimate corresponding to data in the simulate table with simulate_index equal to the sample index. This requires running number_sample fits of the model variables.

asymptotic
If method is asymptotic, The asymptotic statics of the model variables is used to generate number_sample samples of the model variables The samples with different values of sample_index are independent. (Note that the fit_var_table is an additional input in this case.)

simulate_table
If method is simulate, this command has the extra input simulate_table which was created by the previous simulate_command .

fit_var_table
If method is asymptotic, this command has the extra input fit_var_table which was created by a previous fit_command .

sample_table
A new sample_table is created each time this command is run. It contains samples of the model variables. Hence the number of rows in this table is number_sample times the number of rows in the var_table .

Bounds
The samples are all within the specified bounds, including the bounds on the random effects specified by bound_random .

Example
The files sample_command.py and user_asymptotic.py contain examples and tests using this command.
