bin/run_cmake.sh: User Configuration Options

verbose_makefile
Use 'no' for normal and 'yes' for verbose make output: 
 
verbose_makefile='no'

build_type
Use either 'debug' or 'release' for the type of this build: 
 
build_type='debug'
Note that if dismod_at_prefix ends in dismod_at, run_cmake.sh will use a link from the prefix to dismod_at_prefix.debug or dismod_at_prefix.release depending on the choice build_type .

dismod_at_prefix
Prefix where dismod_at will be installed: 
 
dismod_at_prefix="$HOME/prefix/dismod_at"

Prefixes
Prefixes where the special requirements will be installed: 
 
eigen_prefix="$HOME/prefix/dismod_at/eigen"
ipopt_prefix="$HOME/prefix/dismod_at"
cppad_prefix="$HOME/prefix/dismod_at"
suitesparse_prefix="$HOME/prefix/dismod_at"
Note that the Eigen package is installed in a special prefix so that we can suppress warnings coming from its include files (without suppressing warnings from other include files).

Debug and Release
If dismod_at_prefix ends in dismod_at, all the prefixes must be the same, except for the Eigen prefix which has an extra sub-directory at the end. In this case, bin/run_cmake.sh will switch between a debug and release version of dismod_at, depending on build_type . The debug version will have an extra .debug and the release version will have an extra .release at the end of the prefix. In addition, a soft link will be used to map the prefix to either the debug or release version. Note that the same technique will be used to map the build directory to the debug or release version. Also note that if you are using both a debug and release, both versions of the special requirements will need to be installed.

python3_executable
Path to the python3 executable on this machine: 
 
python3_executable='/usr/bin/python3'
An alternative might be '/usr/local/anaconda3-current/bin/python'.

Choosing C++ Compiler
Which c++ compiler should cmake use (empty means cmake will choose it). 
 
cmake_cxx_compiler=''

extra_cxx_flags
Extra C++ flags used during compilation: 
 
extra_cxx_flags='-std=c++11 -Wpedantic -Wall -Wshadow -Wconversion'
An alternative might be '-Wall'.

cmake_libdir
Sub-directory of each prefix where libraries are installed. 
 
cmake_libdir='lib64'

checkpoint_newton_step
If yes, cppad_mixed will use checkpointing during calculation of second derivatives. This affects optimization when quasi_fixed is false, and the asymptotic sampling method. For these computations, checkpointing should use less memory, but take more time. 
 
checkpoint_newton_step='yes'

system_specific_library_list
List of libraries that are needed for a particular system. For example, if when you build dismod_at the pthread library is required by your system, then include it here. Libraries in the list can be separated by spaces or semi-colons. 
 
system_specific_library_list=''
