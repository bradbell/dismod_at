Requirement
One must call replace_like before calling this function.

Log-likelihood
We use y_i to denote the meas_value corresponding to this subset_id . The log-likelihood computed by like_one is the mapping @[@ \ell (u, \theta) = C + \log [ \B{p} ( y_i | u , \theta ) ] @]@ where @(@ u @)@ are the random effects, @(@ \theta @)@ are the fixed effects, and @(@ C @)@ is a constant that does not depend on @(@ ( u , \theta ) @)@.

data_object
This object has prototype
     data_model data_object
 see data_object constructor . It is effectively const (some internal arrays are used for temporary work space and kept between calls to avoid re-allocating memory).

Float
The type Float must be double or a1_double .

subset_id
This argument has prototype
     size_t subset_id
 and is the subset_id we are computing the weighted residual and log-likelihood for.

pack_vec
This argument has prototype
     const CppAD::vector<Float>& pack_vec
 and is all the model_variables in the order specified by pack_info ; i.e., @(@ (u , \theta) @)@. Only the pack_info_mulcov_meas subvectors of pack_vec are used by like_one (note that other components of @(@ (u, \theta ) @)@ are used to compute avg documented below).

avg
This argument has prototype
     const Float& avg
 and is the average integrand , @(@ A_i ( u , \theta ) @)@, for the specified data point. This can be calculated using data_model_average .

delta_out
The input value of delta_out does not matter. Upon return it is the adjusted standard deviation delta for this data point.

residual
The return value has prototype
     residual_struct<Float> residual
 see residual_struct . It contains the weighted residual and the corresponding log-density.

index
Note that the index field is set to the value subset_id .

Example
The file like_one_xam.cpp contains an example and test of using this routine.
